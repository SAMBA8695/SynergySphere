# About

SynergySphere is a project collaboration platform designed to streamline team workflows and boost productivity. It offers a robust API for project management and a user-friendly frontend for seamless interaction.

*   **Effortless Project Tracking:** Manage projects, tasks, and team assignments from a centralized dashboard.
*   **Enhanced Team Collaboration:** Facilitate communication and task delegation within your team.

## Built With

*   **Backend:** NestJS, Prisma
*   **Frontend:** Next.js, React

## Getting Started

Follow these steps to set up your development environment and run SynergySphere.

### Prerequisites

*   **Node.js:** Version 18.18.0 or higher (check `frontend/package-lock.json` for specific requirements).
*   **npm:** Comes bundled with Node.js.

### Installation

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/your-username/synergy-sphere.git
    cd synergy-sphere
    ```

2.  **Install backend dependencies:**
    ```bash
    cd backend
    npm install
    ```

3.  **Install frontend dependencies:**
    ```bash
    cd frontend
    npm install
    ```

### Running the Application

1.  **Start the backend server:**
    ```bash
    cd backend
    npm start
    ```
    The backend will typically run on port 3000.

2.  **Start the frontend development server:**
    ```bash
    cd frontend
    npm run dev
    ```
    The frontend will typically run on port 3001 and be accessible via your browser.

## Usage

SynergySphere streamlines team collaboration through intuitive project management.

*   **Create Projects:** Easily set up new projects, define goals, and assign owners.
*   **Manage Members:** Invite team members to projects, assign roles, and manage permissions.
*   **Collaborate on Tasks:** Break down projects into actionable tasks, assign them to team members, and track progress.
*   **Real-time Updates:** Stay informed with live notifications on task assignments, status changes, and project milestones.

## API

SynergySphere offers a RESTful API for managing projects.

*   **Create Project:**
    *   `POST /projects`
    *   **Request Body Example:**
        ```json
        {
          "name": "New Project",
          "description": "A project to test API functionality"
        }
        ```
    *   **Response Example:**
        ```json
        {
          "id": "123e4567-e89b-12d3-a456-426614174000",
          "name": "New Project",
          "description": "A project to test API functionality",
          "createdAt": "2023-10-27T10:00:00Z"
        }
        ```

*   **Get Project by ID:**
    *   `GET /projects/{projectId}`
    *   **Example:** `GET /projects/123e4567-e89b-12d3-a456-426614174000`
    *   **Response Example:** (Same as Create Project response)

*   **Update Project:**
    *   `PUT /projects/{projectId}`
    *   **Request Body Example:**
        ```json
        {
          "description": "Updated project description"
        }
        ```
    *   **Response Example:** (Updated project object)
